# Contributing

## Ziel
Dieses Repo beschreibt ein professionelles Dev-Agent-Betriebssystem. Änderungen sollen den Standard schärfen, nicht verwässern.

## Änderungsprinzipien

- Änderungen müssen einen klaren Nutzen haben
- Keine unnötige Komplexität hinzufügen
- Standards explizit machen statt implizit lassen
- Vorlagen so gestalten, dass sie in echten Projekten benutzbar bleiben
- Security, Qualität und Betrieb immer mitdenken

## Bevor du etwas änderst

Prüfe:
- welches Problem wird gelöst?
- ist die Änderung dauerhaft hilfreich?
- macht sie das System klarer oder nur größer?
- braucht es eine neue Datei oder reicht eine präzise Ergänzung?

## Arten von sinnvollen Beiträgen

- Rollenprofile schärfen
- neue Gates oder Checklisten ergänzen
- Stack-spezifische Standards dokumentieren
- bessere Übergabe-Templates hinzufügen
- Praxis-Learnings aus echten Projekten einarbeiten
- Anti-Patterns dokumentieren

## Pull-/Change-Standard

Jede relevante Änderung sollte idealerweise enthalten:
- Ziel der Änderung
- betroffene Rollen / Prozesse
- neue Standards oder Anpassungen
- Auswirkungen auf bestehende Abläufe

## Nicht erwünscht

- Buzzword-Erweiterungen ohne Praxisnutzen
- Dopplungen von bereits dokumentierten Regeln
- unklare Verantwortlichkeiten
- Sicherheitsaspekte ohne belastbare Aussage
