# Example Project Lifecycle

## Beispiel
Ein kleines SaaS-Dashboard mit Login, Nutzerrollen und Admin-Bereich.

## Schritt 1 — Lead
- Ziel definieren
- Scope begrenzen
- Erfolgskriterien festlegen
- Project Brief erstellen

## Schritt 2 — Research
- Auth-Optionen vergleichen
- Datenbank-/Backend-Ansatz bewerten
- Hosting-Optionen prüfen
- Risiken benennen

## Schritt 3 — Architect
- Datenmodell definieren
- Rollen und Rechte festlegen
- Seiten- und API-Struktur planen
- Definition of Done festlegen

## Schritt 4 — Design/UI
- zentrale User-Flows skizzieren
- Dashboard-Struktur planen
- Form- und Status-Komponenten standardisieren

## Schritt 5 — Build
- Auth
- Dashboard-Grundlayout
- zentrale API-Flows
- Admin-Schutz
- Doku / Env / Migrations

## Schritt 6 — QA
- Login-/Logout-Flows prüfen
- Rollenwechsel / Rechte prüfen
- Edge Cases und Fehlerzustände prüfen

## Schritt 7 — Security
- Autorisierung prüfen
- Admin-Zugriffe prüfen
- sensible Daten / Logs prüfen
- Secret-Handling prüfen

## Schritt 8 — Deploy/Ops
- Env setzen
- Build/Deploy testen
- Monitoring vorbereiten
- Release Checklist finalisieren

## Schritt 9 — Post Release
- Risiken beobachten
- bekannte Lücken dokumentieren
- nächste Verbesserungen priorisieren
