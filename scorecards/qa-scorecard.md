# QA Scorecard

## Ziel
Eine schnelle, klare Bewertung der Umsetzungsqualität.

## Bewertungsskala
- 0 = nicht vorhanden / unzureichend
- 1 = schwach
- 2 = brauchbar
- 3 = gut
- 4 = stark

## Kriterien

### 1. Functional Correctness
Erfüllt die Umsetzung den Scope und die Akzeptanzkriterien?

### 2. Edge Cases
Sind relevante Randfälle bedacht?

### 3. Maintainability
Ist die Struktur verständlich und wartbar?

### 4. UX / Flow Quality
Ist das Verhalten für Nutzer klar und sinnvoll?

### 5. Error Handling
Werden Fehler sauber behandelt?

### 6. Documentation Fit
Sind relevante Doku/Notizen aktualisiert?

## Ergebnis
- 20-24 = Stark
- 14-19 = Solide, aber mit Verbesserungsbedarf
- 8-13 = Fragil
- 0-7 = Nicht release-reif

## Pflichtnotiz
Neben der Zahl immer die 3 wichtigsten Schwächen benennen.
